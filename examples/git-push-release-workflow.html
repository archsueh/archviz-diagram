1. 准备阶段

脚本执行严格 frontmatter 校验 + 文件检查，然后打印 dry-run 摘要并等待确认。这是 G0-G6 门控的体现。

2. Git 操作（核心）

git push origin main --tags 是整个流程的决定性推送。它将本地 tag 与变更同步到 GitHub。没有这一步，release 无法完成。

  • git add -A
  • git commit（干净则跳过）
  • git tag -f v0.2.5
  • git push ... --tags
