# 信息可视化图表类型调研报告

**项目**: archviz-skills  
**目的**: 为理性极简克制的信息可视化技能包（不限于 Mermaid）提供主流图表类型参考，支持图标、图表、流程、数据可视化等。  
**风格约束**: 遵循 archviz-skills DESIGN.md —— Swiss 网格、暖纸质感 + 单 IKB 锚点、结构优于装饰、预览兼容（Mermaid + ASCII 优先）、反 slop、最小墨水原则。  
**数据来源**: Data Visualisation Catalogue、Stephen Few《Show Me the Numbers》、Edward Tufte、Shneiderman 数据-任务分类、Atlassian/Infogram 等主流资源（2026 年最新）。  
**范围**: 主流 2D 图表（适合信息设计），排除纯 3D archviz 渲染（另行处理）。重点推荐适合克制风格的类型。

## 1. 分类框架（基于 Few + Shneiderman）

Stephen Few 将图表按**数据关系**分类（7 大类），Shneiderman 按**数据类型**分类（6 大类）。结合使用，便于技能包的类型选择 + 预检查。

### Few 的 7 种关系（推荐优先）
1. **排名 (Ranking)**: 比较类别大小。
2. **时间序列 (Time Series)**: 随时间变化。
3. **部分-整体 (Part-to-Whole)**: 组成比例。
4. **偏差 (Deviation)**: 与参考值的差异。
5. **分布 (Distribution)**: 数据分散情况。
6. **相关 (Correlation)**: 两个变量关系。
7. **地理空间 (Geospatial)**: 位置相关。

### Shneiderman 的 6 种数据类型
- 多变量 (Multivariate)
- 空间 (Spatial)
- 时间 (Temporal)
- 层次 (Hierarchical)
- 网络 (Network)
- 文本 (Text)

**技能包设计原则**:
- 每种类型必须有 **Brief + Dials + Tokens** 检查。
- 优先 ASCII/Mermaid 兼容源（避免 <br/>、长中文标签溢出）。
- 图标支持：为类别、节点、流程提供极简线条图标（与 shape vocabulary 对齐）。
- 变体规则：同一概念在文档中多次出现时，使用不同视觉处理（避免同质化）。
- 反默认：简单比较用表格而非饼图；避免彩虹色、3D、chartjunk。

## 2. 主流图表类型清单（按 Few 关系分类）

### 2.1 排名 (Comparison / Ranking)
- **Bar Chart / Column Chart** (条形图/柱状图)
  - 用途: 比较离散类别数值。
  - 推荐场景: 类别数量 < 10-15。
  - 克制用法: 水平条形（长标签友好）、单色 + 细线、排序后显示。
  - Mermaid/ASCII: flowchart 或 bar (xychart-beta)、ASCII |===|。
  - 图标: 类别前加极简符号。
  - 避免: 3D 柱、过多颜色。
- **Radar / Spider Chart** (雷达图)
  - 用途: 多维度比较（如性能指标）。
  - 推荐: 少于 5-7 轴。
  - 克制: 少填充、清晰网格。
- **Bullet Graph** (子弹图，Few 推荐)
  - 用途: 目标 vs 实际 + 范围（KPI）。
  - 极简首选，替代仪表盘。

### 2.2 时间序列 (Temporal / Time Series)
- **Line Chart** (折线图)
  - 用途: 趋势、变化。
  - 推荐: 多线时用颜色区分 + 直接标签。
- **Area Chart** (面积图，含堆叠)
  - 用途: 累积趋势。
  - 克制: 避免过多堆叠（用 Stream Graph 变体）。
- **Gantt Chart** (甘特图)
  - 用途: 项目时间线、任务进度（本技能包已有强支持）。
  - 规则: 内部用短代码 (V1.1)，外部表格 + ASCII 兜底。
- **Candlestick / OHLC** (K 线图)
  - 用途: 金融时间序列。
- **Calendar Heatmap** (日历热力图)
  - 用途: 时间模式（如活跃度）。

### 2.3 部分-整体 (Composition / Part-to-Whole)
- **Pie / Donut Chart** (饼图/环形图)
  - **强烈不推荐**（Few/Tufte 经典批评：角度判断差、比较难）。
  - 替代: Stacked Bar 或 Treemap。
- **Stacked Bar / 100% Stacked Bar** (堆叠柱状图)
  - 用途: 组成 + 比较。
- **Treemap** (树状图)
  - 用途: 层次组成，大小编码面积。
  - 克制: 浅色填充 + 清晰边框、少层级。
- **Sunburst / Icicle** (旭日图/冰柱图)
  - 用途: 层次部分-整体。
- **Waffle Chart** (华夫饼图)
  - 极简替代饼图。

### 2.4 分布 (Distribution)
- **Histogram** (直方图)
  - 用途: 数值分布。
- **Box Plot / Box & Whisker** (箱线图)
  - 用途: 中位数、四分位、异常值（Few 基础四之一）。
- **Violin Plot** (小提琴图)
  - 结合密度 + 箱线。
- **Density Plot / KDE** (密度图)
- **Dot Plot / Strip Plot** (点图)
  - 极简分布展示。

### 2.5 相关 / 关系 (Relationship / Correlation)
- **Scatter Plot** (散点图)
  - 用途: 两个连续变量关系（Few 基础）。
  - 增强: 加趋势线、颜色编码第三变量（Bubble Chart）。
- **Heatmap** (热力图)
  - 用途: 矩阵相关、密度。
  - 克制: 顺序色阶（非彩虹）、清晰标签。
- **Bubble Chart** (气泡图)
- **Network Diagram / Graph** (网络图)
  - 用途: 关系、流程（用 Chord Diagram 变体）。
- **Parallel Coordinates** (平行坐标)
  - 多维关系。
- **Sankey Diagram** (桑基图)
  - 用途: 流量、能量流（本技能包已有支持）。
  - 规则: 加权边、语义标签、ASCII 兜底。

### 2.6 地理空间 (Geospatial)
- **Choropleth Map** (等值区域图)
  - 用途: 区域数据（颜色编码）。
- **Bubble Map / Dot Map** (气泡地图)
- **Flow Map / Connection Map** (流向地图)
- **Cartogram** (变形地图，面积编码）

### 2.7 层次结构 (Hierarchical)
- **Tree Diagram / Dendrogram** (树状图/系统树)
- **Circle Packing** (圆填充)
- **Treemap / Sunburst** (见上)

### 2.8 其他实用类型（信息设计常用）
- **Flowchart / Process Diagram** (流程图)
  - 用途: 步骤、决策（Mermaid 强项）。
- **Timeline** (时间轴)
  - 结合 Gantt 或自定义。
- **Bullet / KPI Card** (KPI 卡片)
- **Word Cloud** (词云) —— **避免**（位置编码差，改用 Bar 或 Treemap）。
- **Illustration Diagram** (示意图) —— 结合图标使用。
- **Error Bars** (误差棒) —— 分布增强。

## 3. 图标在信息可视化中的作用（新增重点）

用户需求包含“各种信息可视化图标”。
- **类别图标**: 代表不同组（极简线条图标，单色，符合 shape vocabulary）。
- **流程/节点图标**: Start/End/Decision（ASCII/Mermaid 兼容）。
- **数据图标**: 数量、趋势符号（小多重、sparkline）。
- **设计原则** (克制):
  - 统一网格、对齐、少细节。
  - 与主图表 tokens 一致（暖纸底 + IKB 线）。
  - ASCII 兜底 + SVG 输出。
  - 预检查: 图标数量 ≤ 图标槽位，避免视觉噪声。

推荐基础图标集（后续技能实现）:
- 节点: 圆/方/菱形
- 箭头: 实线/虚线/加粗（语义区分）
- 状态: 完成/进行中/警告（极简符号）

## 4. 选择指南（用于技能包预检查）

1. **数据类型** → 匹配 Few/Shneiderman 框架。
2. **任务** → 比较用 Bar，趋势用 Line，关系用 Scatter。
3. **受众** → 克制风格：清晰 > 花哨。学术/报告用精确（Scatter/Box），演示用简单（Bar/Line）。
4. **输出模式**:
   - Mermaid: 流程、Sankey、Gantt、简单图。
   - ASCII: 终端/Gantt/流程兜底。
   - 图标增强: 所有类型。
5. **避免**:
   - 饼图（除非 2-3 部分）。
   - 3D、过多颜色、装饰。
   - 长标签直接进 bar（用代码 + 表格，如 Gantt 规则）。

## 5. 推荐技能包覆盖优先级

**核心（已有基础）**:
- Bar/Column, Line, Scatter, Gantt, Sankey, Heatmap, Treemap。

**扩展（信息设计特色）**:
- Box/Violin (分布), Bullet (KPI), Flowchart (过程), Icon-enhanced versions。
- 图标系统独立 skill。
- 组合可视化（Dashboard 级，但克制）。

**未来**:
- 网络图、地图（如果需要地理）。
- 文本 viz（但优先表格）。
